# d28.9 — Clear a param's Session/Global tier value

**Bead:** billz-d28.9 (P4, task) · **Date:** 2026-07-12 · **Status:** design approved

## Problem

d28.4's tier stores (`sessionParams`, `globalParams`) only ever **write**
(`setSessionParams`/`setGlobalParams` merge; there is no delete). Consequences:

1. A Global default persists in `localStorage` forever — no in-app way to remove
   it.
2. A Session value shadows a Global one (Session > Global by §5 precedence) with
   no way to clear the Session value except an app restart.

Both are correct per the precedence model but leave stale shared values
unremovable in-app.

## Goal

Add an in-app affordance to clear a param's **Session** or **Global** value, with
a cleared Global value removed from `localStorage`.

## Scope

**In:** an × button on the inherited-value badge in the param bar that clears the
**surfaced** tier's value for that param; store `clear*` functions; a cleared
Global persists its removal.

**Out:** clearing a shared value that is **shadowed** by a higher tier (a Global
under a Local value has no badge, so no ×) — a rare case; a full shared-params
manager is a separate feature. No change to Local values (those are per-query,
cleared by editing the field / re-running). No Rust change.

## Design

The inherited badge (`↳ session` / `↳ global`, `ParamBar.svelte`) already renders
only when the displayed value comes from a shared tier (i.e. no Local value
shadows it). Making it actionable is the minimal, self-documenting affordance:
"this value comes from Session — × to drop it." Clearing Session re-surfaces
Global (chain to clear both).

### Store clear functions (mirror the existing setters)

```ts
// sessionParams.svelte.ts — in-memory; delete is reactive (Svelte 5 proxy).
export function clearSessionParam(name: string): void {
  delete sessionParams[name];
}
```

```ts
// globalParams.svelte.ts — delete + re-persist (mirrors setGlobalParams).
export function clearGlobalParam(name: string): void {
  delete globalParams[name];
  try {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(globalParams));
  } catch (e) {
    console.warn("billz: failed to persist global params to localStorage", e);
  }
}
```

`delete` on a `$state` proxy fires the `deleteProperty` trap, so the reactive
`paramSources` derived (`App.svelte`, reads the stores directly) recomputes and
the badge updates/flips immediately.

### App handler — clear + refresh the field (`App.svelte`)

```ts
function onClearTier(name: string, tier: "session" | "global") {
  if (tier === "session") clearSessionParam(name);
  else clearGlobalParam(name);
  // Reflect the new reality: the field falls to the next resolved value (a lower
  // tier or empty). Reads the stores AFTER the clear.
  const param = curParams.find((p) => p.name === name);
  if (param) paramValues[name] = resolve(param, sessionParams, globalParams) ?? "";
}
```

Clearing `↳ global` on `@today` removes it from `localStorage` **and** empties the
field (nothing lower); clearing `↳ session` falls the field to Global if present.
The `paramValues` `$effect` reads the stores untracked, so the clear does not
re-run it — only this explicit `paramValues[name]` write updates the field (no
loop, no fight). `resolve` is added to `App.svelte`'s `paramBarLogic` import.

### ParamBar — an × button on each inherited badge (`ParamBar.svelte`)

Add an `onClearTier?: (name: string, tier: "session" | "global") => void` prop
(defaulting to a no-op, like `onScopeChange`), and render a × after each badge:

```svelte
{#if sources[p.name] === "session"}
  <span class="inh session" title="value inherited from the Session tier">↳ session</span>
  <button class="clear-tier" aria-label="Clear Session value for {p.name}"
          title="Clear this Session value" onclick={() => onClearTier(p.name, "session")}>×</button>
{:else if sources[p.name] === "global"}
  <span class="inh global" title="value inherited from the Global tier">↳ global</span>
  <button class="clear-tier" aria-label="Clear Global value for {p.name}"
          title="Clear this Global value" onclick={() => onClearTier(p.name, "global")}>×</button>
{/if}
```

A `.clear-tier` style (small, muted, hover-emphasized) keeps it unobtrusive.

## Testing

- **No new unit-testable pure logic** — the `clear*` fns mutate runes stores (like
  the existing, untested `set*Params`); the delete+persist mirrors
  `setGlobalParams`. Verification is gates + manual.
- **Gates:** `just ui-check` (svelte-check 0 errors/0 warnings), `just ui-test`,
  `just ui-build`.
- **Manual smoke:** set `@today` to Global (scope Global, value, Run) → reopen a
  query using `@today` → field pre-fills from `↳ global`. Click × → field empties,
  badge gone; reload the app → the Global value is gone (removed from
  localStorage). For Session>Global: with both set, `↳ session` shows; × → falls to
  `↳ global`; × again → empty.

## Acceptance criteria

- A param showing `↳ session` / `↳ global` has an × that clears that tier's value
  for the param; the field falls to the next resolved value.
- A cleared **Global** value is removed from `localStorage` (gone after reload).
- Local values and other params are unaffected; no Rust change.
- `cargo fmt`/`clippy` clean (no Rust); `just verify` green; no `mssql_client` in
  `app`/`core` API (backend untouched).
