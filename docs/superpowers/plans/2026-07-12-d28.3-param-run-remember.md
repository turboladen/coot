# d28.3 — Parameterized run + remember-last-value (Local) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run a saved query with its `@param`s filled in, remembering the values (Local scope) so a repeat run is one click.

**Architecture:** A new `run_params` Tauri command delegates to the already-tested `core::run_with_params`. The frontend derives `@param`s from a saved query's SQL, shows a fields-only **param bar** in the tab, and the existing toolbar Run becomes param-aware — on run it remembers values into `Param.last_value` (via `save_query`) and calls `run_params`.

**Tech Stack:** Rust (core + Tauri `app`), Svelte 5 runes, bun test, `@tauri-apps/api`.

## Global Constraints

- Driver stays behind `core`: no `mssql_client` type in `app` or any public API.
- `cargo fmt` + `cargo clippy` clean; **warnings are errors**. `just verify` green.
- Decimals/money cross JSON as strings (unaffected here).
- bun for all JS tooling; Svelte 5 runes; **no SvelteKit**.
- Scope is **Local only** — no Session/Global stores, no scope selector (that's `d28.4`). A param's field pre-fills from `last_value ?? ""`.
- Raw-text params (`sqlType: null`) must render **visibly unsafe** (§5).

```rust
/// The parameterized run path (d28.3). Like `run_sql` but binds/splices
/// `params` via `run_with_params` and sends a SINGLE batch (no `GO`-splitting —
/// a parameterized query spanning `GO` is out of scope). `database` maps to the
/// `ExecutionContext` exactly as `run_sql`.
#[tauri::command]
async fn run_params(
    id: ConnectionId,
    database: Option<String>,
    sql: String,
    params: Vec<ResolvedParam>,
    state: State<'_, AppState>,
) -> AppResult<Vec<QueryResult>> {
    let cfg = state
        .connections
        .get(&id)?
        .ok_or_else(|| CoreError::Config(format!("no connection {}", id.0)))?;
    let mut ctx = ExecutionContext::new(cfg.id.clone());
    if let Some(db) = database {
        ctx = ctx.with_database(db);
    }
    Ok(billz_core::run_with_params(&cfg, &state.secrets, &ctx, &sql, &params).await?)
}
```

```ts
// Pure, rune-free param logic for the saved-query param bar (d28.3). Unit-tested
// via paramBarLogic.test.ts; ParamBar.svelte / App.svelte are the runes wrappers.
import type { Param, ResolvedParam } from "./api";

// Matches T-SQL param placeholders. NOTE (billz-u2t / PLAN §6): a regex also
// matches @x inside a string literal or comment → a harmless extra field (the
// actual substitution is still d28.2's typed bind / flagged raw-text splice).
// A proper T-SQL lexer would tighten this; deferred with u2t.
const PARAM_RE = /@[A-Za-z_]\w*/g;

// Ordered param list for a query: scan @names in `sql` (dedup, first-appearance
// order) and merge with `stored` — an existing name keeps its sqlType/scope/
// lastValue; a new name defaults to raw-text (null) / local / unset.
export function deriveParams(sql: string, stored: Param[]): Param[] {
  const byName = new Map(stored.map((p) => [p.name, p]));
  const seen = new Set<string>();
  const out: Param[] = [];
  for (const m of sql.matchAll(PARAM_RE)) {
    const name = m[0];
    if (seen.has(name)) continue;
    seen.add(name);
    out.push(byName.get(name) ?? { name, sqlType: null, lastValue: null, scope: "local" });
  }
  return out;
}

// Execute-time params from the derived params + the bar's current field values
// (keyed by param name). Missing value → "" (an unset field).
export function toResolvedParams(params: Param[], values: Record<string, string>): ResolvedParam[] {
  return params.map((p) => ({ name: p.name, sqlType: p.sqlType, value: values[p.name] ?? "" }));
}

// Remember (Local, d28.3): copy of `params` with each lastValue set to the field
// value; a param with no field value keeps its prior lastValue. d28.4 will route
// writes by scope.
export function rememberValues(params: Param[], values: Record<string, string>): Param[] {
  return params.map((p) => (p.name in values ? { ...p, lastValue: values[p.name] } : p));
}
```
