# d28.7 — Right-click table → run saved query scoped to it — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Right-click a table in the tree → pick a saved query that uses `@table` → it opens in a tab with `@table` set to the node and column-matching params auto-typed, ready to run.

**Architecture:** A context menu on `TableNode` lists saved queries referencing `@table`; picking one calls `openScopedQuery`, which persists the query with `@table`'s value + auto-typed params (via d28.5's `catalogTypeToSqlType`) and opens it in a tab targeting the table's DB — App's existing param bar / `run_params` path does the rest. No new run path.

**Tech Stack:** Svelte 5 runes, bun test.

## Global Constraints

- Builds on merged code: `TableNode.svelte`, `selectTopQuery.ts` (`quoteIdent`), `savedQueries.svelte` (`library`, `save`), `paramBarLogic.ts` (`catalogTypeToSqlType`, `deriveParams`), `api.ts` (`listColumns`, `ColumnInfo.dataType`, `SavedQuery`, `Param`), `tabs.svelte` (`newTabWithContent(content, database?, savedQueryId?)`), the App param bar + `run_params`.
- Scoped-open reuses d28.3–d28.5 entirely; no new run mechanism; no Rust change.
- `@table` is the literal param convention (raw-text identifier); auto-type fills only **unset** (`sqlType: null`) params, never overwrites; `@table` excluded from auto-type.
- Scoped tab targets the **node's** database (`db`), so `[schema].[table]` resolves.

// Saved queries whose SQL references the literal @table param (the ones a table
// right-click can scope). `(?<!@)` skips @@table; `(?![A-Za-z0-9_])` is the word
// boundary so @table2 / @tablename don't count. Case-sensitive: @table is the
// convention that openScopedQuery fills.
export function queriesReferencingTable(queries: SavedQuery[]): SavedQuery[] {
  const re = /(?<!@)@table(?![A-Za-z0-9_])/;
  return queries.filter((q) => re.test(q.sql));
}

// Auto-type params from a table's columns (d28.7, consuming d28.5): for each param
// that is NOT @table, is UNSET (sqlType null), and whose name (minus @, case-
// insensitive) matches a column, set its sqlType via catalogTypeToSqlType (itself
// null for a float/real/etc. column → stays raw-text). Already-typed params, @table,
// and non-matching params are unchanged (manual overrides win).
export function autoTypeParams(params: Param[], columns: ColumnInfo[]): Param[] {
  const byLowerName = new Map(columns.map((c) => [c.name.toLowerCase(), c]));
  return params.map((p) => {
    if (p.name === "@table" || p.sqlType !== null) return p;
    const c = byLowerName.get(p.name.slice(1).toLowerCase());
    return c ? { ...p, sqlType: catalogTypeToSqlType(c.dataType) } : p;
  });
}
```

// d28.7 scoped-open orchestration: run a saved query "scoped" to a right-clicked
// table. Persists the query with @table's value + auto-typed column params, then
// opens it in a tab targeting the TABLE's database (so [schema].[table] resolves).
// App's param bar + run_params path take over — no new run mechanism. Kept out of
// TableNode to keep the tree node focused; .svelte.ts because `save` mutates the
// savedQueries $state store.
import { type SavedQuery, listColumns } from "./api";
import { autoTypeParams, deriveParams } from "./paramBarLogic";
import { save as saveQuery } from "./savedQueries.svelte";
import { newTabWithContent } from "./tabs.svelte";
import { quoteIdent } from "./tree/selectTopQuery";

export async function openScopedQuery(
  connId: string,
  db: string,
  schema: string,
  table: string,
  query: SavedQuery,
): Promise<void> {
  const columns = await listColumns(connId, db, schema, table);
  const tableRef = `${quoteIdent(schema)}.${quoteIdent(table)}`;
  const derived = deriveParams(query.sql, query.params);
  const typed = autoTypeParams(derived, columns);
  const params = typed.map((p) => (p.name === "@table" ? { ...p, lastValue: tableRef } : p));
  await saveQuery({ ...query, params });
  // Target the table's DB (not query.targetDatabase) so @table resolves.
  newTabWithContent(query.sql, db, query.id);
}
```
